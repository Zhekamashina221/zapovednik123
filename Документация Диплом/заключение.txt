ЗАКЛЮЧЕНИЕ

В рамках дипломного проекта разработан геоинформационный веб-портал «Zapovednik», предназначенный для интерактивного представления объектов природного наследия Республики Беларусь (ООПТ). Краткая формулировка задачи: на основе анализа предметной области и существующих информационных ресурсов спроектировать и реализовать программное средство, обеспечивающее каталог ООПТ, отображение объектов на карте, поиск и фильтрацию, просмотр подробных сведений об объекте, построение маршрутов следования, работу зарегистрированных пользователей (избранное, отзывы, сохранённые маршруты) и администрирование контента; выполнить тестирование, оформить пояснительную записку и обосновать экономическую целесообразность разработки.

Степень соответствия проектных решений заданию оценивается как высокая. Реализованы основные функции, предусмотренные темой и разделами анализа: клиентское одностраничное приложение, серверное REST API, реляционное хранение данных, интерактивная карта с кластеризацией меток, список и карточка ООПТ, подборки маршрутов, учётная запись с подтверждением электронной почты, личный кабинет и панель администратора. Отклонения от максимально возможного объёма задания носят организационно-технический характер и не отменяют практической ценности продукта. Не реализованы отдельное мобильное приложение и полноценный автономный режим работы без сети Интернет; построение геометрии маршрута выполняется через внешний сервис OpenRouteService, что создаёт зависимость от доступности API и наличия ключа. Каталог наполняется локальной базой данных, а не непрерывной синхронизацией с единым государственным реестром ООПТ в режиме реального времени — для учебного проекта принят компромисс: устойчивость демонстрации и контролируемое наполнение данных при развёртывании на учебном стенде.

Пути решения задачи определялись последовательным прохождением этапов жизненного цикла программного обеспечения. На этапе анализа изучены официальные и отраслевые источники сведений об ООПТ, сформулированы функциональные и нефункциональные требования, построены UML-диаграммы (вариантов использования, деятельности, классов). На этапе проектирования выбрана архитектура «клиент — сервер»: Vue.js 3 на стороне клиента, Node.js и Express на стороне сервера, SQLite для хранения данных. На этапе реализации разработаны модули каталога и карты, карточки ООПТ, маршрутизации, учётной записи, профиля и администрирования; на этапе тестирования проверены основные сценарии работы гостя, пользователя и администратора. Экономический раздел подтвердил обоснованность затрат: объём функций оценён по каталогу приложения Б (22 850 LOC), трудоёмкость — по приложению В (879 ч), плановая себестоимость — 2958,07 руб., отпускная цена с НДС — 4082,14 руб.

Нетрадиционные и усиленные по сравнению с простым справочником способы решения включают следующие решения. Для отображения большого числа объектов на карте применена кластеризация меток (Leaflet.markercluster), что улучшает читаемость интерфейса при масштабировании. Вместо самостоятельной реализации маршрутного движка использовано подключение к OpenRouteService с возвратом клиенту GeoJSON, расстояния и длительности — это сократило срок разработки и повысило точность маршрутов. Предусмотрены тематические подборки маршрутов по нескольким ООПТ, экспорт построенного маршрута в форматы GPX и GeoJSON, генерация QR-кода для размещения на информационных стендах, режим «рядом со мной» при фильтрации каталога. Для разграничения доступа применена схема JWT с refresh-токенами и обязательным подтверждением email перед использованием персональных функций.

Использованные методы и средства: объектно-ориентированный анализ и проектирование (UML); проектирование REST API; одностраничная архитектура (SPA); реляционная модель данных; методы модульного и компонентного построения интерфейса. Средства разработки: языки JavaScript (ES-модули), фреймворк Vue.js 3, сборщик Vite, маршрутизатор Vue Router, хранилище Pinia, библиотеки Leaflet и Leaflet.markercluster, серверная платформа Node.js, фреймворк Express, СУБД SQLite (better-sqlite3), средства аутентификации (bcrypt, JWT), загрузка файлов (multer), среда разработки Visual Studio Code, система контроля версий Git. При оформлении документации и расчётах использованы методические материалы по оценке объёма ПО (приложения Б и В) и нормативные положения по охране труда и калькуляции себестоимости.

Программный продукт допускает дальнейшую модификацию. Целесообразно перенести хранение данных на PostgreSQL или иную серверную СУБД при промышленной эксплуатации; реализовать регламентное обновление каталога из официальных открытых данных; добавить поддержку белорусского и английского языков интерфейса; разработить прогрессивное веб-приложение (PWA) с кэшированием карт и справочников для частичной работы без сети; расширить аналитику посещений в административной панели; внедрить полнотекстовый поиск и подписку на новые объекты в выбранном регионе; при необходимости создать мобильное приложение на базе общего API.

Таким образом, поставленная в дипломном проекте задача решена: создано работоспособное программное средство «Zapovednik», отвечающее целям информирования пользователей об ООПТ Республики Беларусь и удобного планирования посещения объектов. Пояснительная записка содержит описание анализа, проектирования, реализации, тестирования, охраны труда и экономических расчётов; представленные материалы позволяют оценить достигнутые результаты и подтвердить выполнение дипломного проекта.
