5 Экономический раздел

Экономика представляет собой науку, изучающую распределение ограниченных ресурсов для удовлетворения потребностей общества в настоящем и будущем. В связи с этим реализация программного обеспечения требует не только технического, но и экономического обоснования.

Экономический раздел дипломного проекта направлен на оценку затрат и определение целесообразности разработки геоинформационного портала «Zapovednik» — интерактивной карты природного наследия Республики Беларусь. Цель экономического анализа — расчёт совокупных затрат на создание программного обеспечения, а также определение возможного экономического эффекта от его применения. Экономическая эффективность может выражаться в сокращении времени поиска сведений об ООПТ, повышении доступности каталога для граждан и специалистов, а также в потенциальной выгоде разработчика при передаче проекта заказчику (учебному заведению, государственному учреждению или туристической организации).

В состав экономического раздела входят:
определение структуры (этапов) работ по созданию программного обеспечения;
определение объёма функций программного обеспечения;
составление сметы затрат на разработку программного обеспечения;
определение отпускной цены программного обеспечения.

Приведённые расчёты и оценки подтверждают экономическую обоснованность проекта «Zapovednik» и демонстрируют возможность его дальнейшего развития и внедрения.


Определение объёма функций программного обеспечения

Оценка объёма программного обеспечения выполняется метрикой LOC (строки исходного кода) на основе стандартного каталога функций, приведённого в приложении Б. В каталоге (таблица Б.1) для каждого кода функции указаны нормативные объёмы в строках при разработке на разных языках. Для портала «Zapovednik» (клиентская часть на JavaScript, серверная на Node.js) в качестве базы для расчёта принят столбец Java как наиболее близкий по типу технологий.

Порядок расчёта следующий:
просмотреть каталог приложения Б и отобрать коды функций, которые реализованы в проекте «Zapovednik»;
для каждого отобранного кода взять фиксированное значение LOC из столбца Java таблицы Б.1 (не подсчитывать строки в файлах проекта);

Пример: код 101 «Организация ввода информации» в столбце Java имеет 130 строк; если в проекте есть ввод (формы входа, поиск, фильтры), к итогу прибавляется 130.

Статья «Электроэнергия» (Э). Затраты на электроэнергию находятся исходя из продолжительности периода разработки программного обеспечения, количества кВт·ч, затраченных на проектирование, и тарифа за 1 кВт·ч, в соответствии с формулой 5.1:

Э = Кэ × Тр,						(5.1)

где Кэ — стоимость одного кВт·ч, руб.;

Тариф на электроэнергию для прочих потребителей составляет 0,3265 руб. за 1 кВт·ч. Время разработки проекта — 2 месяца, среднее потребление энергии в месяц составило 18 кВт·ч, то есть было потреблено 36 кВт·ч. Исходя из вышеизложенного, на электроэнергию было затрачено:

Э = 0,3265 × 36 = 11,75 (руб.)

Статья «Заработная плата» (З). Заработная плата представляет собой денежное выражение стоимости труда сотрудника, определяемое уровнем квалификации, сложностью выполняемых функций, а также объёмом и качеством выполненной работы. В Республике Беларусь при формировании системы оплаты труда применяется механизм, основанный на базовой ставке и тарифном коэффициенте в соответствии с Единой тарифной сеткой (ЕТС).

Для должности разработчика программного обеспечения принят тарифный коэффициент 1,21, что соответствует четвёртому квалификационному разряду. Базовая тарифная ставка 1-го разряда (ТС1) составляет 297 рублей (с 1 января 2026 г.). Тарифная ставка специалиста определяется по формуле 5.2:

ТСi = ТС1 × ТК,						(5.2)

где ТС1 — тарифная ставка специалиста 1-го разряда, руб.;
ТК — тарифный коэффициент для программистов.

Тарифная ставка работника равна:

ТС4 = 297 × 1,21 = 359,37 (руб.)

Для перевода месячной тарифной ставки в часовую используется нормативный месячный фонд рабочего времени 164 ч:

ТСч = ТС4 / 164 = 359,37 / 164 = 2,19 (руб./ч)

Заработная плата (З) по проекту рассчитывается по формуле 5.3 с учётом нормативной трудоёмкости из приложения В:

З = ТСч × t,						(5.3)

где ТСч — часовая тарифная ставка, руб./ч;
t — нормативная трудоёмкость разработки, ч (по приложению В).

З = 2,19 × 879 = 1925,01 (руб.)

Статья «Отчисления на социальные нужды» (Зсз). Отчисления определяются по нормативу в процентном отношении к фонду заработной платы исполнителей. К данным затратам относят отчисления в Фонд социальной защиты населения и на обязательное страхование в Белгосстрах.

Согласно законодательству, норматив отчислений в Фонд социальной защиты населения — 34 %, отчислений на обязательное страхование от несчастных случаев в Белгосстрах — 0,1 % в зависимости от отрасли предприятия. Сумма отчислений рассчитывается по формуле 5.4:

Зсз = З × Нсз / 100 %,					(5.4)

где З — заработная плата по проекту, руб.;
Нсз — суммарный норматив отчислений (34,1 %).

Зсз = 1925,01 × 34,1 % / 100 % = 656,43 (руб.)

Заказчик возмещает разработчику полную стоимость проекта, включая ожидаемую прибыль. Так как программное обеспечение создаётся для конкретного заказчика (учебного заведения или организации-партнёра), чистая прибыль после налогообложения может рассматриваться как экономический эффект для разработчика от реализации проекта.

Отпускная цена — не цена единицы тиражируемого продукта, а стоимость проекта в целом, включая исходные коды и пояснительную документацию.

Для расчёта отпускной цены необходимо рассчитать прибыль, закладываемую в цену. Норму рентабельности примем на уровне 15 %, так как данное программное средство является рисковым капитальным вложением. Прибыль рассчитывается по формуле 5.8:

П = С × Нр / 100 %,					(5.8)

где С — общая сумма расходов по смете, руб.;
Нр — норма рентабельности.

П = 2958,07 × 15 % / 100 % = 443,71 (руб.)

Отпускная цена рассчитывается по формуле 5.9:

ОЦ = С + П,						(5.9)

где С — плановая себестоимость, руб.;
П — прибыль, руб.

ОЦ = 2958,07 + 443,71 = 3401,78 (руб.)

Рассчитаем стоимость проекта с учётом налога на добавленную стоимость. Ставка НДС установлена в размере 20 %. НДС рассчитывается по формуле 5.10:

НДС = ОЦ × Сндс / 100 %,				(5.10)

где ОЦ — отпускная цена, руб.;
Сндс — ставка НДС.

НДС = 3401,78 × 20 % / 100 % = 680,36 (руб.)

Цена программного обеспечения с учётом НДС представляет собой сумму отпускной цены и НДС и рассчитывается по формуле 5.11:

ОЦсНДС = ОЦ + НДС,					(5.11)

где ОЦ — отпускная цена, руб.;
НДС — налог на добавленную стоимость, руб.

ОЦсНДС = 3328,12 + 665,62 = 3993,74 (руб.)

Разработчик портала «Zapovednik» может передать заказчику проект по цене 3993 рубля 74 копейки с учётом НДС, что покроет затраты на разработку и обеспечит запланированную прибыль.
